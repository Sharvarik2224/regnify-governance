# n8n signature + audit flow (Regnify HR)

This mirrors your onboarding pipeline and adds signature traceability.

## 1) Fetch HR signature metadata
- Add an **HTTP Request** node before PDF generation.
- Method: `GET`
- URL: `{{ $env.BACKEND_BASE_URL }}/api/signatures/{{ $json.hr_id }}`
- Response JSON fields: `signature.file_url`, `signature.file_hash`

## 2) Download signature image
- Add **HTTP Request** node.
- URL: `{{ $('Get Signature').item.json.signature.file_url }}`
- Response format: `File`

## 3) Add signature and audit footer into PDF payload
- Before `Multiple offer letters`, append these payload keys:
  - `signature_image_url`
  - `signature_hash`
  - `audit_footer`: `Digitally generated by Regnify HR`
  - `audit_timestamp`: `{{ $now }}`
  - `document_id`: `RGN-OL-{{ $now.format('YYYY') }}-{{ $json.employee_id }}`
- Update PDFMonkey template to render signature at bottom and footer metadata.

## 4) Generate document hash after PDF creation
Use a **Code** node after PDF generation:

```js
const crypto = require('crypto');

const pdfBase64 = $binary?.data?.data;
if (!pdfBase64) {
  throw new Error('Missing PDF binary payload at $binary.data.data');
}

const documentHash = crypto
  .createHash('sha256')
  .update(Buffer.from(pdfBase64, 'base64'))
  .digest('hex');

return [{
  json: {
    document_hash: documentHash,
    document_id: $('Prepare Payload').item.json.document_id,
    signature_hash: $('Get Signature').item.json.signature.file_hash,
    signed_by: $('Prepare Payload').item.json.hr_id,
    signed_at: new Date().toISOString(),
    workflow_id: $workflow.id,
  }
}];
```

## 5) Persist audit record
Option A (preferred in this repo):
- Use **HTTP Request** node to `POST {{ $env.VITE_API_BASE_URL }}/api/document-audit`
- Body fields: `document_id`, `signature_hash`, `document_hash`, `signed_by`, `signed_at`, `workflow_id`

Option B:
- Use MongoDB node to insert directly into `document_audit`.
